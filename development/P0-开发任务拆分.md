## 1. 拆分目标

基于 P0.1 范围冻结稿和架构设计，将工单系统 P0.1 拆解为可开发、可验证、可迭代交付的任务集合。

P0.1 核心闭环：

```text
问题单创建/编辑/查看 -> 人工分流 -> 创建三类工单 -> 来源追溯 -> 叶子工单执行状态机 -> 工单列表 -> 基础统计
```

---

## 2. 开发阶段划分

| 阶段 | 名称 | 目标 |
|---|---|---|
| DEV-00 | 工程与基础设施 | 搭建项目结构、数据库、认证基础、通用规范 |
| DEV-01 | 基础数据模型 | 完成核心表、枚举、迁移和基础仓储 |
| DEV-02 | 问题单模块 | 创建、编辑、列表、详情、关闭 |
| DEV-03 | 工单基础模块 | 三类工单创建、详情、编辑、来源追溯 |
| DEV-04 | 分流模块 | 问题单转业务需求、技术需求、缺陷 |
| DEV-05 | 状态机模块 | 分配、开始、进度、完成、取消 |
| DEV-06 | 列表与基础筛选 | 问题单列表、工单列表、基础筛选 |
| DEV-07 | 基础统计 | 基础总览、分类统计、状态分布 |
| DEV-08 | 权限与审计 | 角色权限、数据范围、操作日志补齐 |
| DEV-09 | 集成验收 | 端到端用例、异常用例、回归测试 |

## 3. DEV-00 工程与基础设施

### T-00-01 项目结构初始化

目标：建立前后端或单体项目的基础结构。

任务：

- 创建项目目录。
- 配置环境变量。
- 配置代码规范。
- 配置 lint / format。
- 配置基础测试框架。
- 配置本地开发启动脚本。

验收：

- 项目可本地启动。
- 基础测试可运行。
- 代码检查命令可运行。

### T-00-02 数据库连接与迁移机制

任务：

- 配置数据库连接。
- 建立 migration 目录和脚本。
- 支持本地初始化数据库。
- 支持回滚或重建开发数据库。

验收：

- 可执行数据库迁移。
- 本地数据库可初始化核心表。

### T-00-03 通用 API 响应和错误码

任务：

- 定义统一响应结构。
- 定义统一错误结构。
- 定义 P0 错误码。
- 定义请求 ID 或 trace ID。

验收：

- 成功响应格式一致。
- 失败响应格式一致。
- 参数错误、无权限、状态非法等错误码可返回。

## 5. DEV-02 问题单模块

### T-02-01 创建问题单 API

接口：

```http
POST /api/issues
```

任务：

- 校验标题、描述。
- 设置默认状态 pending_triage。
- 记录创建人、提交人、创建时间。
- 写入 issue_status_log。
- 写入 audit_log。

验收：

- 标题/描述为空时返回校验错误。
- 创建成功后状态为待分流。
- 不创建工单。

### T-02-02 编辑问题单 API

接口：

```http
PUT /api/issues/{issueId}
```

任务：

- 校验权限。
- 校验状态限制。
- 支持编辑基础字段。
- 已转工单时提示或标记不会同步工单。
- 已关闭问题单禁止编辑。
- 记录变更日志。

验收：

- 待分流问题单可编辑。
- 已关闭问题单不可编辑。
- 编辑不改变状态，不创建工单。

### T-02-03 问题单列表 API

接口：

```http
GET /api/issues
```

任务：

- 支持分页。
- 支持关键词、状态、线索类型、优先级、提交人、创建时间筛选。
- 返回关联工单数。
- 应用数据权限。

验收：

- 问题提交人只能看自己范围。
- 分流处理人可看授权范围。
- 筛选结果正确。

### T-02-04 问题单详情 API

接口：

```http
GET /api/issues/{issueId}
```

任务：

- 返回基础信息。
- 返回状态日志。
- 返回关联工单列表。
- 返回关闭信息。
- 应用数据权限。

验收：

- 已转工单问题单可看到关联工单。
- 已关闭问题单可看到关闭原因。

### T-02-05 关闭问题单 API

接口：

```http
POST /api/issues/{issueId}/close
```

任务：

- 校验状态 pending_triage。
- 校验关闭权限。
- 校验关闭原因。
- 更新状态 closed。
- 写入关闭人、关闭时间。
- 写入状态日志和审计日志。

验收：

- 关闭原因必填。
- 关闭后从待分流列表移出。
- 不创建工单。

### T-02-06 问题单前端页面

页面：

- 问题单列表。
- 问题单详情。
- 新建问题单。
- 编辑问题单。
- 关闭问题单弹窗。

验收：

- 用户可完成创建、编辑、查看、关闭闭环。

## 6. DEV-03 工单基础模块

### T-03-01 手动创建工单 API

接口：

```http
POST /api/work-items
```

任务：

- 支持 type=business_requirement / technical_requirement / defect。
- 校验标题、描述。
- 设置 source_type=manual。
- 根据执行人/团队计算初始状态。
- 默认 is_leaf=true、level=1。
- 写入初始状态日志和审计日志。

验收：

- 可创建三类工单。
- 未指定执行主体为待分配。
- 指定执行主体为待开发。

### T-03-02 工单编辑 API

接口：

```http
PUT /api/work-items/{workItemId}
```

任务：

- 支持编辑标题、描述、优先级、负责人、执行人、团队、截止时间、扩展字段。
- 禁止修改 type、source_type、status、progress、parent_id。
- 已完成/已取消默认不可编辑执行字段。
- 记录审计日志。

验收：

- 可编辑基础字段。
- 禁止修改不可编辑字段。

### T-03-03 工单列表 API

接口：

```http
GET /api/work-items
```

任务：

- 支持分页。
- 支持关键词、分类、状态、来源类型、优先级、负责人、执行人、团队、是否叶子筛选。
- 应用数据权限。

验收：

- 列表展示三类工单。
- 父工单字段预留但 P0 基本为空。

### T-03-04 工单详情 API

接口：

```http
GET /api/work-items/{workItemId}
```

任务：

- 返回基础字段。
- 返回分类扩展字段。
- 返回来源信息。
- 返回状态日志。
- 返回进度日志。
- 返回关联问题单。

验收：

- 问题单转入的工单可看到来源问题单。
- 手动创建工单展示人为创建来源。

### T-03-05 工单前端页面

页面：

- 工单列表。
- 工单详情。
- 新建业务需求。
- 新建技术需求。
- 新建缺陷。
- 编辑工单。

验收：

- 用户可完成三类工单创建、编辑、查看。

## 7. DEV-04 分流模块

### T-04-01 问题单转业务需求 API

接口：

```http
POST /api/issues/{issueId}/triage/business-requirement
```

任务：

- 校验问题单 pending_triage。
- 校验分流权限。
- 创建业务需求工单。
- 创建来源关系 converted。
- 更新问题单状态 converted。
- 写入日志。
- 保证事务一致性和幂等。

验收：

- 成功后问题单已转工单。
- 工单来源类型为 issue_converted。
- 来源关系可追溯。

### T-04-02 问题单转技术需求 API

同 T-04-01，目标工单类型为 technical_requirement。

### T-04-03 问题单转缺陷 API

同 T-04-01，目标工单类型为 defect。

### T-04-04 分流前端交互

任务：

- 问题单详情展示分流操作区。
- 转业务需求 / 技术需求 / 缺陷时展示草稿表单。
- 草稿默认继承问题单标题、描述、优先级等字段。
- 用户确认后调用分流 API。

验收：

- 用户可从问题单详情完成转三类工单。
- 重复点击不重复创建。

## 8. DEV-05 状态机模块

### T-05-01 状态机服务实现

任务：

- 实现工单状态流转校验。
- 实现状态动作统一入口。
- 实现状态日志写入。
- 实现状态非法错误码。

验收：

- 非法状态流转会被拒绝。
- 合法状态流转写入日志。

### T-05-02 分配工单

接口：

```http
POST /api/work-items/{workItemId}/assign
```

任务：

- 校验叶子工单。
- 校验状态 unassigned。
- 校验执行人或团队。
- 更新状态 ready_for_dev。

验收：

- 待分配可分配。
- 非待分配不可重复分配。

### T-05-03 开始处理

接口：

```http
POST /api/work-items/{workItemId}/start
```

任务：

- 校验 ready_for_dev。
- 更新状态 in_progress。
- 进度可默认设为 1。

验收：

- 待开发可进入开发中。

### T-05-04 更新进度

接口：

```http
POST /api/work-items/{workItemId}/progress
```

任务：

- 校验 in_progress。
- 校验进度 1-99。
- 写入进度日志。

验收：

- 0、100、非法值被拒绝。
- 已完成/已取消不可更新。

### T-05-05 完成工单

接口：

```http
POST /api/work-items/{workItemId}/complete
```

任务：

- 校验 in_progress。
- 状态设为 completed。
- progress=100。
- completed_at=now。

验收：

- 完成后状态和进度正确。

### T-05-06 取消工单

接口：

```http
POST /api/work-items/{workItemId}/cancel
```

任务：

- 校验可取消状态。
- 校验取消原因。
- 状态设为 canceled。
- canceled_at=now。

验收：

- 取消原因必填。
- 已完成默认不可取消。

### T-05-07 状态机前端动作

任务：

- 工单详情展示分配、开始、进度、完成、取消按钮。
- 根据状态和权限控制按钮展示。
- 操作成功刷新详情。

验收：

- 用户可在详情页完成状态闭环。

## 12. DEV-09 集成验收

### T-09-01 端到端主链路用例

用例：

1. 创建问题单。
2. 编辑问题单。
3. 转业务需求。
4. 查看来源关系。
5. 分配工单。
6. 开始处理。
7. 更新进度。
8. 完成工单。
9. 查看统计变化。

### T-09-02 三类工单链路用例

- 问题单转业务需求。
- 问题单转技术需求。
- 问题单转缺陷。
- 手动创建业务需求。
- 手动创建技术需求。
- 手动创建缺陷。

### T-09-03 异常用例

- 关闭原因为空。
- 标题/描述为空。
- 非待分流问题单重复分流。
- 来源关系创建失败回滚。
- 非法状态流转。
- 非叶子工单操作，P1 后验证。
- 无权限访问。

## 13. 推荐开发顺序

```text
DEV-00 -> DEV-01 -> DEV-02 -> DEV-03 -> DEV-04 -> DEV-05 -> DEV-06 -> DEV-07 -> DEV-08 -> DEV-09
```

更适合并行的方式：

| 并行线 | 任务 |
|---|---|
| 后端 A | 数据模型、问题单、分流、工单 |
| 后端 B | 状态机、统计、权限 |
| 前端 A | 问题单页面、分流交互 |
| 前端 B | 工单页面、状态动作、统计页 |
| 测试 | E2E 用例、异常用例、权限用例 |
