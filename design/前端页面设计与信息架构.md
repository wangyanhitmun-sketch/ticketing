## 1. 设计目标

本信息架构面向 P0.1，目标是支撑最小闭环：

```text
问题单输入 -> 待分流问题池 -> 人工分流 -> 三类工单 -> 叶子工单执行 -> 基础统计
```

P0 页面重点不是复杂看板，而是让用户高效完成：

- 创建和编辑问题单。
- 在待分流问题池中处理问题单。
- 将问题单转为业务需求、技术需求、缺陷或关闭。
- 创建、查看、编辑和推进工单。
- 查看基础统计。

## 4. 工作台

### 4.1 页面目标

让用户进入系统后快速看到自己需要处理的事项。

### 4.2 信息结构

```text
工作台
├── 指标卡片
│   ├── 待分流问题单
│   ├── 我的待开发工单
│   ├── 我的开发中工单
│   └── 已逾期工单（P1）
├── 快捷入口
│   ├── 新建问题单
│   ├── 新建业务需求
│   ├── 新建技术需求
│   └── 新建缺陷
├── 我的工单列表
└── 待分流问题池摘要（有分流权限时展示）
```

### 4.3 P0 交互

- 点击指标卡片跳转到对应列表。
- 点击快捷入口进入创建页。
- 点击工单进入工单详情。
- 点击问题单进入问题单详情。

## 5. 问题单列表 / 待分流问题池

### 5.1 页面目标

支持查看、筛选和进入问题单详情，分流处理人可集中处理待分流问题。

### 5.2 列表字段

| 字段 | P0.1 | 说明 |
|---|---|---|
| 问题单编号 | 是 | 唯一编号 |
| 标题 | 是 | 问题摘要 |
| 线索类型 | 是 | 需求线索、缺陷线索、未知 |
| 状态 | 是 | 待分流、已转工单、已关闭 |
| 优先级 | 是 | P0-P3 |
| 分类 | 是 | 问题分类 |
| 提交人 | 是 | 问题提交人 |
| 来源渠道 | 是 | 手动创建等 |
| 创建时间 | 是 | 默认倒序 |
| 更新时间 | 是 | 最近修改时间 |
| 关联工单数 | 是 | 已转工单后展示 |

### 5.3 筛选区

P0.1：

- 关键词。
- 状态。
- 线索类型。
- 优先级。
- 提交人。
- 创建时间。

P0.2/P1：

- 标签。
- 是否有关联工单。
- 分流处理人。
- 外部编号。

### 5.4 操作

| 操作 | P0.1 | 说明 |
|---|---|---|
| 新建问题单 | 是 | 跳转新建页 |

| 转业务需求 | 是 | 详情页执行 |
| 转技术需求 | 是 | 详情页执行 |
| 转缺陷 | 是 | 详情页执行 |
| 关闭 | 是 | 详情页执行 |
| 批量导入 | P0.2 | 后置 |

## 6. 问题单详情

### 6.1 页面目标

展示问题单完整上下文、当前状态和分流操作。

### 6.2 页面布局

```text
问题单详情
├── 顶部标题区
│   ├── 编号 + 标题
│   ├── 状态 Badge
│   ├── 优先级
│   └── 操作按钮
├── 基础信息区
│   ├── 线索类型
│   ├── 分类
│   ├── 来源渠道
│   ├── 提交人
│   ├── 创建/更新时间
│   └── 关闭信息（已关闭时）
├── 描述与补充信息区
│   ├── 描述
│   ├── 影响范围
│   ├── 期望结果
│   ├── 实际结果
│   └── 复现步骤
├── 分流操作区（待分流且有权限）
│   ├── 转业务需求
│   ├── 转技术需求
│   ├── 转缺陷
│   └── 关闭
├── 关联工单区
└── 状态/操作日志区
```

### 6.3 状态化展示

| 状态 | 重点展示 |
|---|---|
| 待分流 | 分流操作区、编辑入口、关闭入口 |
| 已转工单 | 关联工单区、转化时间、来源关系 |
| 已关闭 | 关闭原因、关闭人、关闭时间 |

### 6.4 分流按钮规则

- 仅待分流问题单展示。
- 用户必须具备分流权限。
- 点击后进入对应转工单草稿。

## 7. 转工单草稿页 / 弹窗

### 7.1 页面目标

在问题单转工单前，让用户确认和编辑目标工单内容。

### 7.2 适用动作

- 转业务需求。
- 转技术需求。
- 转缺陷。

### 7.3 信息结构

```text
转工单草稿
├── 左侧/上方：来源问题单摘要
│   ├── 标题
│   ├── 描述
│   ├── 线索类型
│   ├── 优先级
│   └── 影响范围/复现步骤
└── 右侧/下方：目标工单表单
    ├── 工单分类（固定）
    ├── 标题
    ├── 描述
    ├── 优先级
    ├── 负责人
    ├── 执行人/团队
    ├── 截止时间
    └── 分类扩展字段
```

### 7.4 默认映射

- 标题默认继承问题单标题。
- 描述默认继承问题单描述。
- 优先级默认继承问题单优先级。
- 缺陷默认继承期望结果、实际结果、复现步骤。

### 7.5 操作

- 确认创建。
- 取消。
- 返回问题单详情。

### 7.6 成功反馈

- 提示“已创建工单”。
- 问题单状态刷新为已转工单。
- 展示新工单入口。

## 9. 工单详情

### 9.1 页面目标

展示工单信息、来源、状态、进度和可执行动作。

### 9.2 页面布局

```text
工单详情
├── 顶部标题区
│   ├── 编号 + 标题
│   ├── 工单分类 Badge
│   ├── 状态 Badge
│   ├── 优先级
│   └── 操作按钮
├── 基础信息区
│   ├── 来源类型
│   ├── 负责人
│   ├── 执行人/团队
│   ├── 截止时间
│   ├── 创建/更新时间
│   └── 是否叶子工单
├── 分类扩展信息区
│   ├── 业务需求字段
│   ├── 技术需求字段
│   └── 缺陷字段
├── 来源追溯区
│   ├── 来源问题单
│   ├── 人为创建信息
│   └── 来源缺陷/AI 创建（P1）
├── 执行动作区
│   ├── 分配
│   ├── 开始处理
│   ├── 更新进度
│   ├── 完成
│   └── 取消
├── 进度与状态日志区
└── 审计日志区
```

### 9.3 动作展示规则

| 当前状态 | 可展示动作 | 条件 |
|---|---|---|
| 待分配 | 分配、取消 | 叶子工单 + 权限 |
| 待开发 | 开始处理、取消 | 叶子工单 + 权限 |
| 开发中 | 更新进度、完成、取消 | 叶子工单 + 权限 |
| 已完成 | 只读，重新打开 P1 | P1 |
| 已取消 | 只读，重新启用 P1 | P1 |

### 9.4 进度展示

- 待分配 / 待开发：0%。
- 开发中：展示当前进度。
- 已完成：100%。
- 已取消：展示取消状态，不突出进度。

## 10. 新建工单页

### 10.1 页面目标

支持用户直接创建业务需求、技术需求、缺陷。

### 10.2 入口

- 工单列表顶部“新建工单”。
- 工作台快捷入口。
- 分类列表快捷入口。

### 10.3 表单结构

通用字段：

- 工单分类。
- 标题。
- 描述。
- 优先级。
- 负责人。
- 执行人。
- 团队。
- 截止时间。

业务需求扩展：

- 业务分类。
- 验收标准。
- 影响范围。

技术需求扩展：

- 技术分类。
- 完成标准。
- 风险说明。
- 影响范围。

缺陷扩展：

- 严重程度。
- 复现步骤。
- 期望结果。
- 实际结果。
- 影响范围。

### 10.4 状态提示

表单应提示：

- 未指定执行人/团队：创建后状态为待分配。
- 指定执行人/团队：创建后状态为待开发。
