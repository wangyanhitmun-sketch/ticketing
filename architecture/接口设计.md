## 1. 设计目标

本文档定义工单系统 P0.1 的 API 边界，并为 P0.2/P1/P2/P3 预留接口扩展。

P0.1 API 覆盖：

- 问题单创建、编辑、列表、详情、关闭。
- 问题单转业务需求、技术需求、缺陷。
- 工单创建、编辑、列表、详情。
- 叶子工单状态动作：分配、开始、更新进度、完成、取消。
- 基础指标统计。

## 2. 通用约定

### 2.1 响应格式

```json
{
  "success": true,
  "data": {},
  "error": null,
  "requestId": "req_xxx"
}
```

失败响应：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "WORK_ITEM_STATUS_INVALID",
    "message": "当前状态不允许执行该操作",
    "details": {}
  },
  "requestId": "req_xxx"
}
```

### 2.2 分页参数

```text
page: number
pageSize: number
```

### 2.3 排序参数

```text
sortBy: createdAt | updatedAt | priority | dueDate
sortOrder: asc | desc
```

### 2.4 幂等键

写操作可通过请求头传入：

```text
Idempotency-Key: uuid
```

适用场景：

- 问题单转工单。
- 手动创建工单。
- AI 创建确认，P1。

## 3. 错误码

| 错误码 | 说明 |
|---|---|
| UNAUTHORIZED | 未登录 |
| FORBIDDEN | 无权限 |
| VALIDATION_ERROR | 参数校验失败 |
| NOT_FOUND | 对象不存在 |
| CONFLICT | 并发冲突或重复提交 |
| ISSUE_STATUS_INVALID | 问题单状态不允许操作 |
| WORK_ITEM_STATUS_INVALID | 工单状态不允许操作 |
| WORK_ITEM_NOT_LEAF | 工单不是叶子工单 |
| SOURCE_RELATION_FAILED | 来源关系创建失败 |
| INTERNAL_ERROR | 系统异常 |

## 4. 问题单 API

### 4.1 创建问题单

```http
POST /api/issues
```

请求：

```json
{
  "title": "审批流程需要支持多级审批",
  "description": "不同金额走不同审批链路",
  "clueType": "demand_clue",
  "priority": "P1",
  "category": "流程优化",
  "sourceChannel": "manual",
  "impactScope": "审批模块",
  "expectedResult": "支持按金额自动匹配审批链路",
  "actualResult": null,
  "reproduceSteps": null,
  "tags": ["审批", "流程"]
}
```

响应：

```json
{
  "id": "iss_001",
  "issueNo": "ISS-20260529-0001",
  "status": "pending_triage"
}
```

### 4.2 编辑问题单

```http
PUT /api/issues/{issueId}
```

请求：

```json
{
  "title": "审批流程支持多级审批",
  "description": "补充后的描述",
  "clueType": "demand_clue",
  "priority": "P1",
  "category": "流程优化",
  "impactScope": "审批模块"
}
```

### 4.3 问题单列表

```http
GET /api/issues
```

查询参数：

| 参数 | 说明 |
|---|---|
| keyword | 搜索标题、描述、编号 |
| status | pending_triage / converted / closed |
| clueType | demand_clue / defect_clue / unknown |
| priority | P0/P1/P2/P3 |
| submitterId | 提交人 |
| createdFrom | 创建开始时间 |
| createdTo | 创建结束时间 |
| page | 页码 |
| pageSize | 每页数量 |

响应：

```json
{
  "items": [
    {
      "id": "iss_001",
      "issueNo": "ISS-20260529-0001",
      "title": "审批流程支持多级审批",
      "clueType": "demand_clue",
      "status": "pending_triage",
      "priority": "P1",
      "submitterName": "Estelle",
      "createdAt": "2026-05-29T10:00:00+08:00",
      "relatedWorkItemCount": 0
    }
  ],
  "total": 1
}
```

### 4.4 问题单详情

```http
GET /api/issues/{issueId}
```

响应需包含：

- 基础字段。
- 附件。
- 评论，若启用。
- 状态日志。
- 关联工单列表。

### 4.5 关闭问题单

```http
POST /api/issues/{issueId}/close
```

请求：

```json
{
  "closeReasonType": "duplicate",
  "closeReason": "已有同类问题处理"
}
```

响应：

```json
{
  "id": "iss_001",
  "status": "closed",
  "closedAt": "2026-05-29T11:00:00+08:00"
}
```

## 5. 分流 API

### 5.1 问题单转业务需求

```http
POST /api/issues/{issueId}/triage/business-requirement
```

请求：

```json
{
  "title": "审批流程支持多级审批",
  "description": "不同金额走不同审批链路",
  "priority": "P1",
  "ownerId": "user_pm_001",
  "assigneeId": null,
  "teamId": "team_dev_001",
  "dueDate": "2026-06-30",
  "acceptanceCriteria": "可按金额配置审批链路"
}
```

响应：

```json
{
  "issueId": "iss_001",
  "issueStatus": "converted",
  "workItem": {
    "id": "wi_001",
    "workItemNo": "WI-20260529-0001",
    "type": "business_requirement",
    "sourceType": "issue_converted",
    "status": "ready_for_dev"
  }
}
```

### 5.2 问题单转技术需求

```http
POST /api/issues/{issueId}/triage/technical-requirement
```

请求字段在业务需求基础上增加：

```json
{
  "technicalCategory": "performance",
  "completionCriteria": "接口 P95 响应时间小于 500ms",
  "riskNote": "可能影响订单查询模块"
}
```

### 5.3 问题单转缺陷

```http
POST /api/issues/{issueId}/triage/defect
```

请求：

```json
{
  "title": "提交审批时报 500",
  "description": "点击提交审批后页面报错",
  "priority": "P0",
  "severity": "serious",
  "expectedResult": "提交成功并进入审批流",
  "actualResult": "页面返回 500",
  "reproduceSteps": "1. 打开审批页 2. 点击提交",
  "assigneeId": "user_dev_001"
}
```

## 6. 工单 API

### 6.1 创建工单

```http
POST /api/work-items
```

请求：

```json
{
  "type": "business_requirement",
  "title": "新增退款申请入口",
  "description": "用户可在订单页发起退款申请",
  "priority": "P1",
  "ownerId": "user_pm_001",
  "assigneeId": null,
  "teamId": "team_dev_001",
  "dueDate": "2026-06-30",
  "businessCategory": "订单"
}
```

响应：

```json
{
  "id": "wi_001",
  "workItemNo": "WI-20260529-0001",
  "type": "business_requirement",
  "sourceType": "manual",
  "status": "ready_for_dev",
  "isLeaf": true
}
```

### 6.2 编辑工单

```http
PUT /api/work-items/{workItemId}
```

请求：

```json
{
  "title": "新增退款申请入口",
  "description": "补充后的需求说明",
  "priority": "P1",
  "ownerId": "user_pm_001",
  "assigneeId": "user_dev_001",
  "teamId": "team_dev_001",
  "dueDate": "2026-06-30"
}
```

不可通过该接口修改：

- type
- sourceType
- status
- progress
- parentId

### 6.3 工单列表

```http
GET /api/work-items
```

查询参数：

| 参数 | 说明 |
|---|---|
| keyword | 搜索编号、标题、描述 |
| type | business_requirement / technical_requirement / defect |
| status | unassigned / ready_for_dev / in_progress / completed / canceled |
| sourceType | 来源类型 |
| priority | 优先级 |
| ownerId | 负责人 |
| assigneeId | 执行人 |
| teamId | 团队 |
| isLeaf | 是否叶子工单 |
| createdFrom | 创建开始时间 |
| createdTo | 创建结束时间 |
| page | 页码 |
| pageSize | 每页数量 |

### 6.4 工单详情

```http
GET /api/work-items/{workItemId}
```

响应需包含：

- 基础字段。
- 分类扩展字段。
- 来源信息。
- 父子关系，P1。
- 状态日志。
- 进度日志。
- 附件和评论，若启用。

## 7. 工单状态动作 API

### 7.1 分配工单

```http
POST /api/work-items/{workItemId}/assign
```

请求：

```json
{
  "assigneeId": "user_dev_001",
  "teamId": "team_dev_001",
  "note": "分配给研发处理"
}
```

状态：

```text
unassigned -> ready_for_dev
```

### 7.2 开始处理

```http
POST /api/work-items/{workItemId}/start
```

请求：

```json
{
  "note": "开始处理"
}
```

状态：

```text
ready_for_dev -> in_progress
```

### 7.3 更新进度

```http
POST /api/work-items/{workItemId}/progress
```

请求：

```json
{
  "progress": 50,
  "note": "接口开发完成一半"
}
```

校验：

- 工单必须是叶子工单。
- 状态必须是 in_progress。
- progress 必须为 1-99。

### 7.4 完成工单

```http
POST /api/work-items/{workItemId}/complete
```

请求：

```json
{
  "note": "已完成并自测通过"
}
```

状态：

```text
in_progress -> completed
```

副作用：

- progress = 100。
- completed_at = now。

### 7.5 取消工单

```http
POST /api/work-items/{workItemId}/cancel
```

请求：

```json
{
  "cancelReasonType": "scope_changed",
  "cancelReason": "本期范围调整，暂不处理"
}
```

可取消状态：

- unassigned
- ready_for_dev
- in_progress
