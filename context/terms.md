# 术语表 - 工单系统

| 术语 | 定义 |
|---|---|
| 问题单 | 需求线索或缺陷线索的输入对象，用于分流，不直接承载研发执行 |
| 需求线索 | 可能转为业务需求或技术需求的问题输入 |
| 缺陷线索 | 可能转为缺陷工单的问题输入 |
| 工单 | 业务需求、技术需求、缺陷的统一管理对象 |
| 业务需求 | 面向业务流程、用户体验、产品能力变化的工单 |
| 技术需求 | 面向架构、性能、稳定性、安全、工程效率等技术改进的工单 |
| 缺陷 | 已有功能不符合预期、发生错误或异常的工单 |
| 叶子工单 | 没有子工单的工单，是研发执行载体 |
| 父工单 | 已拆分的一级工单，只做聚合、管理和追踪 |
| 子工单 | 父工单拆分出的二级工单，必然是叶子工单 |
| 个人视图 | 用户个人保存的筛选、分组、排序、字段配置 |
| 团队视图 | 团队共享的视图配置，可由个人视图转化而来 |
| AI 创建 | 用户通过自然语言指令，经 AI 生成并确认后创建工单 |
| AI 智能分流 | AI 对问题单分类、转派、相似项进行建议，P3 范围 |
